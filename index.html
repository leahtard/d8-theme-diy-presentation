Theme Libraries 101

Created through the theme’s yml files

THEME.info.yml

``` name: Theme Name type: theme core: 8.x # Regions regions: header: Header content: Content # Scripts & Styles libraries: - THEME/global ```

Note: All references to THEME would be replaced by your theme name.

- Global theme library will be loaded globally. - Namespace your control. - Because this library is being included in THEME.info.yml - But no CSS is defined here. - Further definition in THEME.libraries.yml.

Theme Libraries 101

Further definition of CSS and JS required

THEME.libraries.yml

``` global: version: VERSION css: base: css/base/base.css: {} layout: css/layout/layout.css: {} component: css/components/components.css: {} js: js/modernizr.js: {} ```

continued in right column 

``` slick: version: VERSION css: component: css/components/slick.css: {} js: js/slick.min.js: {} js/slick.config.js: {} dependencies: - core/drupal - core/jquery ```
- Here we define our CSS/JS. - What you did in a single D7 THEME.info file, you now do in two D8 theme yml files. - Define another library, slick - Dependencies, JQuery not loaded globally - Simply adding this here will not load it anywhere - We will come back to how this is used...

Create a Theme Library

Back to the theme library...

THEME.libraries.yml

``` global: version: VERSION css: base: css/base/base.css: {} layout: css/layout/layout.css: {} component: css/components/components.css: {} js: js/modernizr.js: {} ```

continued in right column 

``` slick: version: VERSION css: component: css/components/slick.css: {} js: js/slick.min.js: {} js/slick.config.js: {} dependencies: - core/drupal - core/jquery ```
- When creating a theme library, we are responsible for downloading assets. - Because slick requires JQuery, we also add that as a dependency. - One file left to review: slick.config.js
